# Input
- 用户任务: {user_command}


# Analysis Guidelines
提取其中的隐秘信息，并识别应用场景。

# Output Format
必须仅输出合法的JSON格式，绝不允许输出任何其他解释性文字。
JSON必须包含以下字段：
- app_context: 应用场景（20字以内，纯英文+下划线，如 "forum_register", "e_commerce_checkout", "chat_window"）
- task_target_entities: 任务目标实体（用户当前操作涉及的敏感信息，如 ["收货地址", "手机号"]）
- privacy_concerns: 隐私担忧（可能泄露的隐私类型列表）
- mask_plan: 打码计划（需要打码的敏感信息列表，每个元素包含 text 和 reason）
- need_mask: 是否需要打码（true/false）
